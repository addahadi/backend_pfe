Full Backend API Contract

Complete REST API reference for the Backend PFE Architecture. This document centralizes security standards (JWT), AI-driven assistant endpoints, and the comprehensive construction estimation engine.

🌐 Base URL  http://localhost:5000/api 📦 Format  JSON (application/json) 🔑 Security  JWT Bearer Auth
🔐 Authentication & Security

Our API uses JSON Web Tokens (JWT) for secure identified access.

Required Authorization Header

HeaderFormatRequiredDescription
Authorization "Bearer <access_token>" Required Pass the accessToken returned by Login or Register in every protected request.
ℹ️ Tokens expire after 15 minutes. Use the refreshToken via the /refresh endpoint to obtain a new access token without logging in again.

🧮 Calculation (Stateless Preview)

Runs the calculation engine without persisting any data. The frontend uses this to show live results before the user decides to save.

POST /calculate Run a cost calculation preview 200 400

Accepts the user's selected formula, material configuration, and input field values, then returns the computed quantities, material costs, and grand total. Nothing is saved to the database.

⚠️ Important: The keys of field_values must be the field_id UUIDs returned by /categories/:id/leaf, NOT short variable names like "L" or "l".
Request Body
FieldTypeRequiredDescription
category_id UUID Required The leaf category being estimated.
selected_formula_id UUID Required The formula to use for calculation (from formulas[].formula_id).
selected_config_id UUID | null Optional Material configuration preset. Pass null if not applicable.
field_values Record<UUID, number> Required Map of { field_id → numeric_value }. Must have at least one entry.
Response — 200 OK
FieldTypeDescription
resultsobjectComputed formula output keyed by expression variable (e.g. { "volume": 4.0 }).
material_linesMaterialLine[]Per-material cost breakdown.
total_costnumberGrand total cost across all materials.
Example Request Body
{
  "category_id":         "d4e5f6a7-b8c9-0123-def0-123456789012",
  "selected_formula_id":  "e5f6a7b8-c9d0-1234-ef01-234567890123",
  "selected_config_id":   "c9d0e1f2-a3b4-5678-2345-678901234567",
  "field_values": {
    "f6a7b8c9-d0e1-2345-f012-345678901234": 5,
    "a7b8c9d0-e1f2-3456-0123-456789012345": 4,
    "b8c9d0e1-f2a3-4567-1234-567890123456": 0.2
  }
}
Example Response — 200 OK
{
  "status": "ok",
  "data": {
    "results": {
      "volume": 4.0
    },
    "material_lines": [
      {
        "material_id":           "mat-uuid-001",
        "material_name":         "Portland Cement",
        "material_type":         "PRIMARY",
        "quantity":              1.6,
        "applied_waste":         0.05,
        "quantity_with_waste":   1.68,
        "unit_price_snapshot":   4500,
        "waste_factor_snapshot": 0.05,
        "sub_total":             7560
      }
    ],
    "total_cost": 7560
  }
}
Example Response — 400 Validation Error
{
  "status":  "error",
  "message": "Validation failed",
  "errors": [
    { "field": "category_id", "message": "Invalid UUID format" },
    { "field": "field_values", "message": "field_values must not be empty" }
  ]

🍃 Estimation Leaves

Save or remove individual leaf calculations within an estimation. Each leaf represents one category's cost breakdown.

POST /estimation/save-leaf Save (or update) a leaf calculation 201 400

Persists a leaf calculation result into the project's estimation. If project_details_id is provided, the existing record is updated (and its material lines are replaced). Otherwise, a new leaf record is created. After saving, the estimation's total_budget is automatically recalculated.

ℹ️ Always call POST /calculate first to get results and material_lines, then pass that data here to persist it.
Request Body

category_idUUIDRequiredThe leaf category being saved.
selected_formula_idUUIDRequiredFormula used for the calculation.
selected_config_idUUID | nullOptionalMaterial config used. Pass null if none selected.
project_details_idUUID | nullOptionalIf provided, updates this existing leaf. If null, creates new.
field_valuesRecord<UUID, number>RequiredInput values used (field_id → value). May be empty object for computed-only formulas.
formula_version_snapshotintegerRequiredFormula version from the leaf endpoint — for audit trail.
resultsobjectRequiredComputed outputs from /calculate (e.g. { "volume": 4.0 }).
material_linesMaterialLine[]RequiredMaterial cost lines returned by /calculate (see schema below).
